# HEARTBEAT.md - 定期チェックリスト

## トークン・メモリ監視 🧠

毎ハートビートで実行：
- [ ] session_status で現在のトークン使用率を確認
  - **警告水準**：70% 以上使用されている
  - **危機水準**：90% 以上（セッション終了まで 10% 未満）
- [ ] 警告水準に達したら：memory に要約を追加 → セッションリセット提案
- [ ] 危機水準に達したら：直ちに「セッションリセット必須」と報告

## 自動要約機能（DM セッション継続用） 📝

毎回実行（分離後）：
- [ ] 現在のセッション内で重要なアクション/決定があったか確認
- [ ] あれば、MEMORY.md の **「API・トークン管理の危機と教訓」** の直後に新セクション追加
  - 例：「## DM セッション要約 (2026-02-10)」
  - 重要な約束、決定、学習、進行中のタスク
- [ ] セッションリセット時（月が知らせる）に自動実行

## その他の定期チェック（ローテーション）

### メール・通知チェック（1日1回程度）
- [ ] 未読メッセージ
- [ ] Discord 提及
- [ ] 緊急通知

### カレンダー・スケジュール（朝）
- [ ] 本日のイベント（<2h）
- [ ] 明日の重要イベント

### プロジェクト状態（午後）
- [ ] Git ステータス（未コミット・プッシュ）
- [ ] 進行中のタスク
- [ ] 期限が近いタスク

---

**ルール**：
- 毎ハートビートでトークン確認は **必須**
- 自動要約は **重要内容があるときだけ**（毎回ではない）
- その他は **ローテーション**（毎回全部は API 消費が多い）
- 重要度や状況に応じて柔軟に
